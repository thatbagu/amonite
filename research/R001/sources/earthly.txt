## What It Is
Earthly is "a build automation tool that uses docker containers to enforce build
repeatability." The system targets both local development and CI environments.

"Earthly's implicit caching and parallelism will make your builds repeatable
and fast."

## Earthfile Format
Earthfiles define build processes similarly to Dockerfiles but with enhanced
capabilities. They're named "Earthfile" regardless of location and start with
a VERSION declaration for compatibility management.

Core Structure:
- Base Target & Inheritance: The initial commands (typically FROM and WORKDIR)
  create an implicit base target that all other targets inherit from unless
  they specify their own environment.
- Targets: "Targets are just sets of instructions we can call on from within
  the Earthfile, or when we run Earthly at the command line." Targets are
  invoked via the command line with "earthly +targetname".

Essential Commands:
- FROM: Specifies the Docker image environment.
- COPY: Two variants: standard (from build context) or cross-target
  ("COPY +build/example ." passes artifacts between targets).
- RUN: Executes commands within the build environment.
- SAVE ARTIFACT: Exports build outputs: "SAVE ARTIFACT output/example [/path]"
- SAVE IMAGE: Creates Docker images.

## Key Principles
Isolation & Hermeticity: Only declared dependencies between targets are shared;
otherwise builds remain completely isolated. "The rest is completely isolated."

Deduplication: Shared base targets execute only once despite multiple dependents.

Atomicity: "A build either succeeds completely or it fails altogether," with
outputs written only upon success.

Parallelization: Independent build steps execute in parallel automatically.
